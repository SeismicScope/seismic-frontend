# Level 2: Theme Tokens

Theme tokens are semantic colors that can be swapped based on the active theme. They reference Core Colors but provide a flexible abstraction layer.

**Key Concept**: These tokens are theme-aware, meaning their values can change when switching themes (e.g., admin vs user interface).

## Primary

The primary color represents the main brand or interface accent color.

## Theme Switching

Theme tokens can be switched by applying different data attributes to automatically change the primary color throughout your interface.

### How It Works

```tsx
// Admin theme - Primary uses Iris (purple)
<div data-theme="admin">
  <button className="bg-ds-primary text-ds-primary-on-primary">
    Admin Action
  </button>
</div>

// User theme - Primary uses Mango (orange)
<div data-theme="user">
  <button className="bg-ds-primary text-ds-primary-on-primary">
    User Action
  </button>
</div>
```

**The same code produces different colors based on the active theme!**

### Key Takeaway

**Same component code, different themes!** The theme tokens automatically resolve to the correct colors:

- **Admin theme**: `--ds-theme-primary-default` → `var(--ds-iris-500)` → Purple
- **User theme**: `--ds-theme-primary-default` → `var(--ds-mango-500)` → Orange

This allows you to create flexible, theme-aware components without hardcoding colors.
